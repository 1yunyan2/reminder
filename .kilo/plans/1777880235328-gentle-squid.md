## 问题诊断

### 问题 1：触摸翻页触发情绪（而非翻页）

**日志证据：**
```
触摸事件: 8, 当前视图: 0  (SHORT_NEXT_PAGE → 无操作，正确)
触摸事件: 1, 当前视图: 0  (SHORT_HEAD → 触发 EMO_HAPPY，错误！)
```

**根因：`bsp_touch.c` 中身体键与翻页键串扰**

在 `touch_scan_task` 的第三步（第 493-499 行），翻页键按下时屏蔽身体键：
```c
if (prev_raw || next_raw) {
    h = false;  a = false;  b = false;
}
```

但这只将当前扫描周期的 `h/a/b` 设为 `false`，**并未重置身体按键的消抖状态机**。如果在前面的扫描周期中，由于电容耦合，`btn_head` 已经累积了 `press_count >= PRESS_DEBOUNCE`（进入了 `is_pressed = true` 状态），那么后续周期中：
- `h = false`（被翻页键屏蔽）
- `update_body_btn(&btn_head, false, false, ...)` → 进入释放分支
- 经过 `RELEASE_DEBOUNCE`(3) 个周期后 → 发送 `TOUCH_EVENT_SHORT_HEAD`

**修复方案：** 当翻页键活跃时，将 `prev_raw || next_raw` 也作为 `in_combo` 条件传给 `update_body_btn`，彻底重置身体键状态。

### 问题 2：时间显示只在左侧

**根因：** `s_menu_body` 使用 `LV_LABEL_LONG_WRAP` + `lv_obj_align(LV_ALIGN_CENTER)` 定位。label 宽度为 `BSP_LCD_WIDTH - 24 = 296px`。由于 label 默认左对齐文本，且居中定位后可能只占屏幕的一部分区域。

当前实现将时间和日历信息全部放在**一个 label** 中（`time_page_render_text`），而非左右分栏。

**修复方案：** 保持单 label 但设置 `LV_TEXT_ALIGN_CENTER` 让文本居中显示，视觉效果更好。

### 问题 3：闹钟页面无法翻页/退出

**根因：功能菜单的事件分发逻辑**

在 `UI_VIEW_FUNCTION_MENU` 状态下：
- `SHORT_NEXT_PAGE` → 如果在闹钟页 → `alarm_list_select_next()`（选择闹钟，**不是翻页**）
- `SHORT_PREV_PAGE` → 如果在闹钟页 → `alarm_list_select_prev()`（选择闹钟，**不是翻页**）
- `LONG_NEXT_PAGE` → 如果在闹钟页 → 进入闹钟编辑（**不是退出**）
- `LONG_PREV_PAGE` → 退出菜单（**这是唯一退出方式**）

用户需要**长按前页键**才能退出闹钟页。但日志显示用户主要触发的是事件 8（SHORT_NEXT_PAGE）和事件 10（LONG_NEXT_PAGE），缺少事件 9（LONG_PREV_PAGE）。

**修复方案：** 在闹钟/倒计时页面，当列表已在边界时，短按翻页改为执行页面切换。

### 问题 4：长按进入时钟界面时崩溃（LoadProhibited）

**崩溃栈：**
```
get_prop_core → get_selector_style_prop → lv_obj_get_style_prop →
calc_dynamic_width → lv_obj_refr_size → layout_update_core →
layout_update_core → lv_obj_update_layout → lv_display_refr_timer
```

**EXCVADDR = 0x00000018**（NULL + 24 偏移）= 对 NULL 指针的成员访问

**根因分析：** LVGL 渲染周期中访问了无效对象。最可能的原因：
1. `ensure_menu_panel()` 在锁内创建对象，但创建过程中子对象的 class/style 可能存在中间状态
2. `alarm_page_create()` 和 `countdown_page_create()` 在 `s_menu_panel` 上创建子对象，如果在渲染周期中同时发生对象创建/隐藏，会导致布局计算访问已删除对象
3. `alarm_page_rebuild()` 中 `lv_obj_clean()` 删除所有子对象后立即重建，如果 LVGL 的异步布局更新引用了被删除的对象，会崩溃

**修复方案：**
- 在 `ensure_menu_panel()` 后调用 `lv_obj_invalidate(lv_screen_active())` 强制刷新
- 在 `alarm_page_rebuild()` 中，先标记容器不可见，清理后重建，再标记可见
- 在所有对象创建/销毁操作后调用 `lv_obj_update_layout()` 确保布局同步更新
- 增加 NULL 检查保护

### 问题 5：大部分触摸都会触发情绪

与问题 1 同根因。身体键的消抖状态未随翻页键的屏蔽而重置。

## 修复方案详细

### 修复 1：`bsp_touch.c` — 消除翻页键与身体键串扰

**文件：** `main/bsp/bsp_touch.c`

**改动：** 在 `touch_scan_task` 中，将翻页键活跃状态传入 `update_body_btn` 的 `in_combo` 参数：

```c
// 修改前：
update_body_btn(&btn_head, h, combo_ha || combo_hb || !body_enabled, ...);
update_body_btn(&btn_abdomen, a, combo_ha || combo_ab || !body_enabled, ...);
update_body_btn(&btn_back, b, combo_hb || combo_ab || !body_enabled, ...);

// 修改后：增加 prev_raw || next_raw 条件
bool page_blocking = prev_raw || next_raw;
update_body_btn(&btn_head, h, combo_ha || combo_hb || !body_enabled || page_blocking, ...);
update_body_btn(&btn_abdomen, a, combo_ha || combo_ab || !body_enabled || page_blocking, ...);
update_body_btn(&btn_back, b, combo_hb || combo_ab || !body_enabled || page_blocking, ...);
```

### 修复 2：`ui_port.c` — 修复时间显示布局

**文件：** `main/ui/ui_port.c`

**改动：** 在 `time_page_render_text` 函数中为 `s_menu_body` 设置文本居中对齐：

```c
// 在 ensure_menu_panel() 中创建 s_menu_body 时增加：
lv_obj_set_style_text_align(s_menu_body, LV_TEXT_ALIGN_CENTER, 0);
```

### 修复 3：`ui_port.c` — 改善闹钟/倒计时页面翻页体验

**文件：** `main/ui/ui_port.c`

**改动：** 在 `UI_VIEW_FUNCTION_MENU` 的事件处理中，闹钟页短按翻页增加边界检测：
- 闹钟页：当已在第一个闹钟时，`SHORT_PREV_PAGE` 改为 `ui_page_prev()`；当已在最后一个闹钟时，`SHORT_NEXT_PAGE` 改为 `ui_page_next()`
- 或者更简单的方式：在闹钟/倒计时页面增加一个明确的"退出"操作提示

### 修复 4：`ui_port.c` — 防止 LVGL 崩溃

**文件：** `main/ui/ui_port.c`

**改动：**
1. 在 `ensure_menu_panel()` 末尾添加 `lv_obj_invalidate(scr)`
2. 在 `alarm_page_rebuild()` 中，先隐藏容器，清理后重建，使用 `lv_obj_invalidate` 刷新
3. 在 `ui_function_menu_enter()` 中，确保所有对象创建完成后统一刷新布局
4. 对所有可能为 NULL 的 LVGL 对象增加 NULL 检查

### 修复 5：`ui_port.c` — 功能菜单中 body 触摸忽略

**当前状态：** 在 `UI_VIEW_FUNCTION_MENU` 和 `UI_VIEW_ALARM_EDIT` 的 switch 中，body 触摸事件（SHORT_HEAD/ABDOMEN/BACK/COMBO_*）落入 `default: break;`，不做任何操作。这已经是正确的。
